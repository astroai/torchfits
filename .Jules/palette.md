## 2024-07-11 - Add _repr_html_ methods for Jupyter Notebook accessibility
**Learning:** Jupyter Notebook interfaces default to unformatted string representations for custom objects, which can be difficult to read and completely lack accessibility attributes for screen readers. Using `_repr_html_` with custom CSS variables (like `var(--theme-ui-colors-background, white)`) and ARIA labels creates a seamless, native-feeling, and accessible experience across light/dark themes.
**Action:** When working on data science libraries providing tabular or complex structures (like HDUs), always implement `_repr_html_` using WCAG-compliant contrast (`rgba` colors) and explicit table semantics (`scope="col"`, `scope="row"`) to ensure data is legible and screen-reader friendly in notebook environments.
