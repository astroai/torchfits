## 2024-07-06 - Accessible HTML Representations in Notebooks
**Learning:** Jupyter notebooks rely heavily on custom `_repr_html_` implementations to format rich data structures like FITS headers and tables. We noticed that adding CSS `overflow: auto` to a wrapping container makes it scrollable, but by default, this container isn't accessible to keyboard navigation or screen readers. Also, custom HTML tables often miss the semantic `scope` attributes needed for a screen reader to associate table headers properly.
**Action:** Always add `tabindex="0"` and a descriptive `aria-label` to custom scrollable `<div>` containers (such as those output via `_repr_html_`). Additionally, ensure all `<th>` tags explicitly declare `scope="col"` (or `scope="row"`) to enhance accessibility. This guarantees robust screen reader and keyboard support natively within Jupyter notebooks.
