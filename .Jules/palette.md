## 2026-07-02 - Accessible Scrollable Tables in Jupyter
**Learning:** Scrollable containers in HTML (often using `overflow: auto`) require explicit keyboard support for users relying on screen readers or keyboard navigation. By default, they do not receive focus, making their scrolled content unreachable if there are no inherently focusable elements inside. Tables inside these containers also benefit from semantic markup like `scope="col"` on header cells (`<th>`) to give screen readers proper structural context.
**Action:** When creating custom HTML representations for objects (e.g., using `_repr_html_` in Jupyter Notebooks) that use scrollable `<div>` containers, always add `tabindex="0"` and an appropriate `aria-label`. Additionally, ensure table headers use `scope="col"` (or `scope="row"`) for proper semantic mapping.
